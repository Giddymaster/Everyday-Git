What is Git

Git is a version control system that allows you to manage and keep track of your source code history. It is a distributed version control system, which means that the entire codebase and its history are available on every developer's computer, which allows for easy branching and merging.A Version Control System (VCS) is a tool that helps developers track changes in their code over time. It enables them to:

  • Keep track of changes made to the code.
  • Collaborate with other developers.
  • Revert to previous versions in case of errors.
  • Work on different features simultaneously without conflicts.
There are other version control systems like Subversion, Mercurial, and Perforce, but Git is the most popular and widely used among developers.

Creating a New Repository

To create a new Git repository, you need to open the terminal, navigate to your project directory, and run the following command:

        git status to check the status of your repository
        git init to initialize a new Git repository

This command will create a new Git repository in your project directory. You can then start tracking your project files using Git.

Adding files to the staging area

After creating a new Git repository, you need to add your project files to the staging area. The staging area is a place where you can group files together before committing them to the repository. This allows you to organize and review changes before they become part of the project history. To add files to the staging area, use the command git add filename for a specific file or git add . to add all files.

Tracked files are files that Git is aware of and is tracking for changes, while untracked files are files that Git is not yet tracking. You can see the status of your files by using the git status command, which will show you which files are tracked, untracked, or staged for commit.

Committing changes

After adding files to the staging area, you need to commit the changes to the repository. A commit is a snapshot of your project at a specific point in time. It records the changes you have made to the project files and stores them in the repository's history. To commit changes, use the command git commit -m "commit message", where the commit message describes the changes you have made.

Branches

Branches in Git are essential for managing different lines of development within a project. They allow developers to work on new features, bug fixes, or experiments in isolation from the main codebase. This means that changes can be made and tested without affecting the stable version of the project. Once the work on a branch is complete and tested, it can be merged back into the main branch, ensuring a smooth and organized workflow.

There are two types of branches in Git:

  1. Local branches - They exist on your local machine and are used for development work.
  2. Remote branches - They exist on a remote repository on GitHub and are used for collaboration and sharing code with other developers.

Creating a branch

To create a new branch in Git, you can use the command git branch branchname. This will create a new branch with the specified name.

Viewing all branches

To view all branches in a Git repository, you can use the command git branch. This will list all the branches in the repository and highlight the active branch with an asterisk (*) next to its name.

Switching active branches

To switch to a different branch, you can use the command git checkout branchname or git switch branchname. This will change the active branch to the specified branch.

Merging a branch

To merge changes from one branch into another, you first need to switch to the branch you want to merge into. Then, use the command git merge branchname, where branchname is the name of the branch you want to merge from.

Deleting a branch

To delete a branch that is no longer needed, you can use the command git branch -d branchname for a local branch or git push origin --delete branchname for a remote branch. This will remove the specified branch from your repository. After merging workout branch, we can now delete it, as its of no use.

Next Steps

Now that you have a good understanding of the basics of Git, here are some next steps you can take to further enhance your skills:

  • Learn about GitHub: GitHub is a web-based platform that uses Git for version control. It provides additional features for collaboration, such as pull requests, issues, and project management tools. You can start by creating a GitHub account and exploring public repositories.
  • Explore advanced Git commands: There are many advanced Git commands and techniques that can help you manage your projects more effectively. Some topics to explore include rebasing, cherry-picking, and stashing.
  • Understand Git workflows: Different teams use different workflows to manage their projects. Some common workflows include Git Flow, GitHub Flow, and GitLab Flow. Learning about these workflows can help you choose the best one for your team.
  • Practice collaboration: Collaborate with other developers on open-source projects or within your team. This will help you get comfortable with branching, merging, and resolving conflicts.
  • Automate tasks with Git hooks: Git hooks are scripts that run automatically in response to certain events in the Git lifecycle. They can be used to automate tasks such as running tests, formatting code, or sending notifications.

Git is a powerful tool for collaboration, allowing multiple developers to work on the same project simultaneously. It provides features like branching, merging, and pull requests that make it easy to manage and review changes made by different team members. Here are some common Git commands used for collaboration:

  • git clone - Clone a remote repository to your local machine.
  • git pull - Fetch changes from a remote repository and merge them into your local branch.
  • git push - Push changes from your local branch to a remote repository.
  • git fetch - Download changes from a remote repository without merging them into your local branch.
  • git merge - Merge changes from one branch into another.
  • git pull request - Create a pull request to propose changes to a repository.

By using these commands, developers can work together on a project, review each other's code, and ensure that changes are integrated smoothly into the codebase.
